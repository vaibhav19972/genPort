iOS / Swift BLE CoreBluetooth

Real-time BLE Proximity App

A proximity-based dating application that leverages Bluetooth Low Energy (BLE) for real-time distance measurement, automated matching triggers, and secure nearby user discovery — all without relying on GPS in indoor environments.

The Challenge

Traditional dating apps rely on GPS for proximity, which is inaccurate indoors and drains battery. The goal was to build a system that detects nearby users within a configurable radius (5m–200m) using BLE signal strength (RSSI), enabling hyper-local, real-time matching in venues like cafés, events, and campuses — with minimal power consumption.

Key Features

  • BLE Radar Interface: A real-time radar-style UI showing nearby users as pulsing dots with live distance indicators updated every 2 seconds.
  • RSSI-to-Distance Calibration: Custom signal propagation model with Kalman filtering for accurate indoor distance estimation from BLE advertisement packets.
  • Automated Proximity Triggers: Configurable distance thresholds that automatically trigger match notifications and profile reveals when two users are within range.
  • Privacy-First Architecture: Randomized BLE advertising UUIDs rotated every 15 minutes. No persistent identifiers broadcast. All profile data exchanged via encrypted server-side channels.
  • Background Scanning: Efficient CoreBluetooth background mode implementation with state restoration, maintaining detection capability while the app is backgrounded.
  • Battery Optimization: Adaptive scan intervals (1s foreground, 5s background) and smart duty-cycling that reduces BLE power consumption by 60% compared to continuous scanning.
